About

はじめまして。AIと一緒にものづくりをしている個人開発者です。

AIと現場の間に立つ個人開発者として、農業と不動産という「デジタル化が遅れた産業」に関わりながら、Claude Code を使った自動化・ツール開発を実践しています。

SubAgents や Skills の設計を通じて、人間の意図をAIの動作に変換する構造設計を学んでいます。コードを書く目的は「仕組みを作ること」——現場の非効率を、再現可能なワークフローに置き換えることが目標です。

技術スタックは Python × Claude Code を軸に、実用性を最優先にした小さく動くものを積み上げています。

// 02 — SUBAGENTS

7つのカスタム SubAgents

.claude/agents/ に配置。@エージェント名で呼び出し、または Claude が自動判断して委譲

🔍@analyzerSTEP 1

Python を静的解析。構造・問題点を HIGH / MEDIUM / LOW で分類してレポート出力

🔨@builderSTEP 2

分析レポートをもとにコードを改善。型ヒント・docstring を自動追加

✅@reviewerSTEP 3

改善コードの網羅性・安全性・一貫性を検証。APPROVED / NEEDS REVISION で最終判定

📋@code-reviewerAgent

コード品質レビュー専門。問題点・改善提案をリスト形式で報告

📝@doc-writerAgent

Python コードを解析して関数・クラスの仕様を README 風 Markdown に自動生成

🧪@test-runnerAgent

正常系・異常系・境界値の unittest を自動生成・実行してサマリーを報告

✍️@seo-blog-writerAgent

SEO 構成で記事を執筆し char-counter スキルで文字数を ±5% 以内に自動最適化

// 03 — SKILLS

11のカスタム Skills

.claude/commands/ に配置。/コマンド名 または自然文で自動発動

コマンド説明カテゴリ
/build-and-reviewAnalyzer→Builder→Reviewer の3段階パイプラインを順次実行Pipeline
/file-summarizerファイル読込→要約→改善提案の3段階処理（引数で文字数指定可）Pipeline
/code-reportPython を静的解析して品質メトリクスを Markdown レポートに出力Analysis
/char-counter改行±・スペース±の4パターンで文字数を正確にカウントUtility
/security-checksOWASP Top 10 準拠でコードの脆弱性を CRITICAL/HIGH/MEDIUM/LOW 分類Security
/git-commit-writergit diff を分析して Conventional Commits 形式のメッセージを自動生成Git
/readme-generatorプロジェクト構成を解析して README.md を自動生成Docs
/performance-guide計算量・メモリ・I/O のボトルネックを特定して改善例を提示Analysis
/database-tipsDBクエリ最適化・インデックス戦略・N+1問題などをアドバイスDB
/changelog-generatorgit log を解析して CHANGELOG.md を自動生成Git
/env-doctor開発環境の設定・依存関係の問題を診断してレポートOps
